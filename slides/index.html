Understanding The Problem

  • List of messages from HL7 stream
  • Data comes from multiple sources
  • Identify all of the unique patients
  • Match to "our" list of patients

Lesson Learned:

What is Record Linking?

Joining the same "real world" data together from different sources.

  • data linkage
  • object identity resolution

Some mesages are very detailed:


{:firstName "BABY GIRL"
 :lastName "fever"
 :ssn "000-00-0000"}
 :visitNumber "888921"

Some messages are sparse:


{:firstName "BABY GIRL"
 :lastName "fever"
 :ssn "000-00-0000"}
 :visitNumber "888921"

Not all in the same format:


{:firstName "BABY GIRL"
 :lastName "fever"
 :ssn "000-00-0000"}
 :visitNumber "888921"

Lesson Learned:

Data Cleaning is Vital

  • Without consistent data, nothing else will work correctly
  • Tends to be time-consuming and tedious
  • Do not skimp here
Expect this to be the longest part of your project

Lesson Learned:

Deterministic Algorithm

  • Good first attempt
  • Keep it simple
  • Think: "secret foreign key"
  • Test the result

Fellegi-Sunter Method

  • Match Weight (m)
  • Unmatch weight (u)

Lesson Learned:

Probabilistic Algorithm

  • Another name: "fuzzy matching"
  • Works surprisingly well
  • Don't overthink the weights - just try something
  • Test the result

Lesson Learned:

Do Approximate String Matching

  • Improves FS results
  • Especially useful for names

Lesson Learned:

Be Creative About Dimensions

  • Can a person have multiple IDs?
  • Is time a useful dimension for matching?
  • What else is unique about your data to help identify?

Lesson Learned:

Build Tools to Visualize

  • Gives you confidence in the outcome
  • Makes edge cases less scary
  • Tools are fun!
