Probabilistic Record Linkage of Hospital Patients

by Chris Oakman

Intro

reveal.js enables you to create beautiful interactive slide decks using HTML. This presentation will show you examples of what it can do.

The Problem

Identify the same patient from multiple data sources.

Lesson Learned:

What is Record Linking?

  • Same "real world" data from difference sources
  • Joining the data together

Lesson Learned:

Data Cleaning is Vital

  • Without consistent data, nothing else will work correctly
  • Tends to be time-consuming and tedious
  • Do not skimp here

Lesson Learned:

Deterministic Algorithm

  • Good first attempt
  • Keep it simple
  • Think: "secret foreign key"
  • Test the result

Fellegi-Sunter Method

  • Match Weight (m)
  • Unmatch weight (u)

Lesson Learned:

Probabilistic Algorithm

  • Another name: "fuzzy matching"
  • Works surprisingly well
  • Don't overthink the weights - just try something
  • Test the result

Lesson Learned:

Do Approximate String Matching

  • Improves FS results
  • Especially useful for names

Lesson Learned:

Be Creative About Dimensions

  • Can a person have multiple IDs?
  • Is time a useful dimension for matching?
  • What else is unique about your data to help identify?

Lesson Learned:

Build Tools to Visualize

  • Gives you confidence in the outcome
  • Makes edge cases less scary
  • Tools are fun!
